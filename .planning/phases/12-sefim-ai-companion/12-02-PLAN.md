    **SefimPulse (sefim-pulse.tsx):**
    - Animated wrapper component using Reanimated v4 (same pattern as SkeletonCard shimmer)
    - Props: `active: boolean`, `children: ReactNode`
    - When active=true: `withRepeat(withSequence(withTiming(1.25, {duration:500}), withTiming(1.0, {duration:500})), -1, false)` on scale + opacity pulse (0.6 to 1.0)
    - When active=false: `withTiming(1)` to reset
    - Uses `useSharedValue`, `useAnimatedStyle`, wraps children in `Animated.View`
    - Export: `SefimPulse`

    **SefimSheet (sefim-sheet.tsx):**
    - Modal-based bottom sheet following IngredientsSheet pattern exactly (Modal, transparent overlay, slide animation, Pressable dismiss)
    - Props:
      ```typescript
      interface SefimSheetProps {
        visible: boolean;
        onClose: () => void;
        chips: SefimQA[];           // pre-loaded Q&A for current step
        messages: SefimMessage[];
        isLoading: boolean;
        onChipTap: (qa: SefimQA) => void;
        onSendQuestion: (text: string) => void;
      }
      ```
    - Layout (top to bottom):
      1. Header: "Sef'im" title + close button (same style as IngredientsSheet header)
      2. Q&A chips row: horizontal ScrollView of chips. Each chip is a Pressable with question text, brand border color (#E07B39). Chips hidden after any has been tapped (show messages instead). If no chips (empty array), skip this section.
      3. Messages area: ScrollView showing conversation. User messages right-aligned (brand bg), assistant messages left-aligned (gray bg). When isLoading=true, show "Sef'im dusunuyor..." with animated dots.
      4. Input row: TextInput + send button (Pressable with MaterialCommunityIcons "send") + mic button (MaterialCommunityIcons "microphone")
    - Voice input handling:
      - Import `ExpoSpeechRecognitionModule` and `useSpeechRecognitionEvent` from `@jamsch/expo-speech-recognition`
      - Mic button: check permission with `ExpoSpeechRecognitionModule.getPermissionsAsync()`. If not granted, request with `requestPermissionsAsync()`. If denied, show Alert "Mikrofon izni gerekli".
      - On permission granted: `ExpoSpeechRecognitionModule.start({ lang: 'tr-TR', interimResults: false })` when mic pressed
      - `useSpeechRecognitionEvent('result', (event) => { setText(event.results[0]?.transcript); })` to capture result
      - `useSpeechRecognitionEvent('end', () => { /* auto-send or let user review */ })` -- put transcribed text in TextInput, user taps send
      - Mic button shows recording state (red tint when active)
    - Brand colors: #E07B39 (orange), #111827 (text), #F3F4F6 (borders), #FEF3EC (light orange bg for chips)
    - Use `expo-haptics` `impactAsync(ImpactFeedbackStyle.Light)` on chip tap for tactile feedback
