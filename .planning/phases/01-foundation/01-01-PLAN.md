  <name>Task 1: Bootstrap Expo project and install dependencies</name>
  <files>package.json, tsconfig.json, app.json, app/_layout.tsx, app/index.tsx, .editorconfig</files>
  <action>
Run `npx create-expo-app@latest TheCook --template default` in the project working directory (C:/Users/dogu1/ClaudeCode/MyProject). This creates the Expo Router template. Then:

1. Install required dependencies:
   ```
   cd TheCook
   npx expo install expo-sqlite
   npm install zod yaml tsx
   npm install --save-dev jest-expo jest @types/jest @testing-library/react-native @types/node
   ```

2. Update `tsconfig.json` to extend `expo/tsconfig.base` and enable strict mode. The relevant section:
   ```json
   {
     "extends": "expo/tsconfig.base",
     "compilerOptions": {
       "strict": true,
       "paths": {
         "@/*": ["./src/*"]
       }
     },
     "include": ["**/*.ts", "**/*.tsx", "scripts/**/*.ts", "__tests__/**/*.ts"]
   }
   ```
   Note: `expo/tsconfig.base` already sets `"noEmit": true` and JSX settings. Strict mode must be explicitly set.

3. Add jest configuration to `package.json`. Add under a top-level `"jest"` key:
   ```json
   {
     "preset": "jest-expo",
     "transformIgnorePatterns": [
       "node_modules/(?!((jest-)?react-native|@react-native(-community)?)|expo(nent)?|@expo(nent)?/.*|@expo-google-fonts/.*|react-navigation|@react-navigation/.*|@unimodules/.*|unimodules|sentry-expo|native-base|react-native-svg)"
     ]
   }
   ```

4. Add scripts to `package.json` `"scripts"` section (merge with existing):
   ```json
   {
     "validate-recipes": "tsx scripts/validate-recipes.ts",
     "build-recipes": "tsx scripts/build-recipes.ts",
     "prebuild": "npm run build-recipes"
   }
   ```
   Note: `prebuild` in package.json scripts runs before `expo prebuild`. This is NOT the same as the existing `expo prebuild` command — do not remove Expo's own prebuild script; rename the Expo one to `"expo:prebuild": "expo prebuild"` if it conflicts.

5. Update `app.json`: Add `"assetBundlePatterns": ["assets/**", "app/assets/**"]` to the `"expo"` config object to ensure `recipes.json` is bundled in production builds.

6. Create `.editorconfig` in the project root:
   ```
   root = true

   [*]
   charset = utf-8
   end_of_line = lf
   indent_style = space
   indent_size = 2
   trim_trailing_whitespace = true
   insert_final_newline = true

   [*.yaml]
   indent_size = 2
   ```
   This prevents Turkish character corruption (Pitfall 2 from research).

7. Update `app/index.tsx` to a minimal placeholder screen:
   ```tsx
   import { Text, View } from "react-native";

   export default function HomeScreen() {
     return (
       <View style={{ flex: 1, alignItems: "center", justifyContent: "center" }}>
         <Text>The Cook — coming soon</Text>
       </View>
     );
   }
   ```

8. Keep `app/_layout.tsx` as the default Stack layout for now (SQLiteProvider wrapping is added in Plan 03). The file should be a clean Expo Router root layout with a Stack navigator and no SQLite imports yet.

Create directories: `src/types/`, `src/db/`, `content/recipes/`, `content/images/`, `scripts/`, `app/assets/`, `__tests__/`.
