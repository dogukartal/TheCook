  <action>
Create `content/AUTHORING-GUIDE.md`. The audience is Hira — a non-developer content author who writes recipes. The guide must explain everything without assuming git knowledge, TypeScript knowledge, or developer tooling familiarity.

The guide must cover:

**Section 1: How recipes are structured**
- Explain that recipes are YAML text files in `content/recipes/`
- Explain that each recipe file = one recipe
- Show the filename convention: `recipe-name.yaml` (kebab-case, Turkish names transliterated: `mercimek-corbasi.yaml`)
- Show a complete minimal example recipe in YAML (can use menemen as the example)

**Section 2: All recipe fields explained**
For every field in the schema, explain in plain Turkish-friendly English:
- What it is
- What values are allowed (list every enum option by name — e.g., for `category`: "ana yemek", "kahvaltı", "çorba", "tatlı", "salata", "aperatif")
- Whether it is required or optional
- An example value

Specific fields to explain clearly:
- `id`: "A unique short name for this recipe. Use the English transliteration, all lowercase, hyphens instead of spaces. Example: `mercimek-corbasi`"
- `allergens`: "A list of allergen tags. Only use these exact values: [list all 14]. Leave as `[]` if the recipe has no common allergens."
- `equipment`: "A list of equipment required. Only use these exact values: [list all enum values]. Leave as `[]` if only basic equipment (a pan and spatula) is needed."
- `ingredientGroups`: "For most recipes, you will have one group with `label: null`. For recipes with separate components (e.g., a dough and a filling), use multiple groups and give each a label (e.g., 'Hamur için', 'İç için')."
- `unit`: "Use only these exact unit values: [list all 9]. Do not make up new units."
- `optional`: "Set to `true` for ingredients the cook can omit. Set to `false` (or leave it out) for required ingredients."
- Each step field: instruction, why, looksLikeWhenDone, commonMistake, recovery — explain what each is for with an example
- `timerSeconds`: "How many seconds this step takes. Leave as `null` if there is no specific timing. Example: `120` for a 2-minute step."
- `stepImage`: "Leave as `null` for now. Developer will add image filenames later."

**Section 3: Writing a new recipe**
Step-by-step workflow:
1. Copy `content/recipes/menemen.yaml` and rename it
2. Edit all fields
3. Run the validator (explain how to open terminal and run `npm run validate-recipes`)
4. Read the error messages and fix them
5. Run again until it says "All recipes valid."
6. Tell the developer the recipe is ready

**Section 4: Understanding validator errors**
Show examples of common error messages and what they mean:
- `Field "steps > 0 > why": Required` — "The 'why' explanation is missing from the first step"
- `Field "ingredientGroups > 0 > items > 0 > unit": Invalid enum value` — "The unit you used is not in the allowed list. Check the units list in Section 2."
- `Field "allergens > 0": Invalid enum value` — "The allergen tag you used is not in the allowed list. Check Section 2."

**Section 5: Images**
- Place cover images in `content/images/` with filename `recipe-id-cover.jpg`
- Place step images in `content/images/` with filename `recipe-id-step-01.jpg`
- Then tell the developer — they will add the filename to the YAML

**Tone:** Friendly and direct. No jargon. Short paragraphs. Use numbered lists for steps. This guide should take a non-technical author 10 minutes to read and leave them confident to write their first recipe.
