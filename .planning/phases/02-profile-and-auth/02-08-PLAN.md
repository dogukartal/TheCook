<objective>
Close the two gaps identified in the Phase 2 verification report.

Gap 1 (Blocker): app.json is missing the iOS URL scheme required for Google OAuth redirect. The @react-native-google-signin/google-signin SDK requires the reversed iOS client ID registered as a CFBundleURLSchemes entry in Info.plist so that Google's OAuth web flow can redirect back to the native app after the user grants consent. Without this, Google Sign In silently fails on iOS.

Gap 2 (Warning): src/db/profile.ts declares `values: unknown[]` in two functions (saveProfile hook and saveProfileToDb standalone). expo-sqlite's runAsync expects SQLiteBindParams (= SQLiteBindValue[]). The mismatch causes two TS2769 errors that fail `npx tsc --noEmit`.

Purpose: Both gaps are isolated single-file fixes. Closing them makes the codebase clean: TypeScript compiles without errors and Google Sign In is correctly configured for iOS.
Output: Updated app.json with ios.infoPlist.CFBundleURLTypes, updated src/db/profile.ts with SQLiteBindValue[] type, clean tsc output.
</objective>
