  <name>Task 1: Create full-screen cooking view with PagerView wiring</name>
  <files>TheCook/app/recipe/cook/[id].tsx</files>
  <action>
    Create `app/recipe/cook/` directory and `[id].tsx` as the full-screen step-by-step cooking view.

    **Screen structure:**
    1. **Top bar** (SafeAreaView top):
       - Back-to-preview button (top-left): MaterialCommunityIcons "close", navigates `router.back()`
       - "Ask Chef" button (top-right): MaterialCommunityIcons "chef-hat". On tap: `Alert.alert("Yakin zamanda!", "Bu ozellik cok yakinda geliyor.")` (coming soon)
       - SegmentedProgressBar below the buttons, spanning full width

    2. **PagerView** (flex: 1, middle area):
       - Import PagerView from `react-native-pager-view`
       - `ref={pagerRef}` for programmatic navigation
       - `initialPage` = session?.currentStep ?? 0 (for resume)
       - `onPageSelected`: update currentStep state, save session to SQLite
       - Guard against mount-fire: use a `mountedRef` to skip the first `onPageSelected` if page === initialPage (Pitfall 3 from RESEARCH.md)
       - Each page renders `<StepContent>` with appropriate props including timer callbacks
       - After the last step page, add one more page for `<CompletionScreen>`

    3. **Bottom navigation bar**:
       - Left: "Malzemeler" button (opens IngredientsSheet)
       - Center/Right: "Geri" (previous step) small text button + "Sonraki" (next step) prominent button
       - On last step, "Sonraki" text changes to "Bitir" and navigates to completion page
       - "Geri" hidden on first step

    4. **Floating TimerIndicator**: positioned absolutely, shows when timer.stepIndex !== currentStep and timer is active

    5. **IngredientsSheet** modal: controlled by `showIngredients` boolean state

    **State management:**
    - `recipe`: loaded via `getRecipeById(db, id)` on mount
    - `currentStep`: number, updated by PagerView onPageSelected
    - `ingredientChecks`: number[], toggled by IngredientsSheet checkbox
    - Timer: `useCookingTimer()` hook
    - `sessionStartedAt`: string, set once on mount (new Date().toISOString()) or from resumed session
    - `showIngredients`: boolean for overlay visibility
    - `showCompletion`: boolean (set true when navigating past last step)

    **Session persistence:**
    - On mount: check `getActiveSession(db)`. If session exists and matches this recipe ID, restore currentStep, ingredientChecks, timer state. If timer was running, recalculate remaining (Pitfall 4 from RESEARCH.md): `newRemaining = timerRemaining - ((Date.now() - timerStartTimestamp) / 1000)`. If <= 0, timer already completed.
    - On every `onPageSelected`: call `saveSession(db, { recipeId, currentStep, timerRemaining, timerStartTimestamp, ingredientChecks, sessionStartedAt })`
    - On ingredient check toggle: save session
    - On timer start/pause: save session with timer state
    - On completion (user reaches completion screen): call `clearSession(db)`

    **Keep-awake:**
    - `import { activateKeepAwakeAsync, deactivateKeepAwake } from 'expo-keep-awake'`
    - Call `activateKeepAwakeAsync()` on mount, `deactivateKeepAwake()` on unmount

    **Timer integration:**
    - When user taps play on a step's CircularTimer: call `timer.start(stepIndex, step.timerSeconds)`
    - When timer completes: visual pulse on CircularTimer (handled by component), no vibration
    - TimerIndicator shows when navigated away from timer step — tapping navigates pagerRef to timer step

    **Navigation:**
    - pagerRef.current?.setPage(index) for prev/next/timer-indicator navigation
    - router.back() for exit (back to preview)
    - router.replace('/(tabs)') for "back to recipes" from completion screen

    **Critical implementation notes:**
    - Do NOT auto-start timers (CONTEXT.md: manual start with play/pause button)
    - One active timer at a time (starting new timer resets previous — handled by useCookingTimer)
    - Sequential navigation only — no step jump list overlay
  </action>
