    2. Create `src/hooks/useCookingTimer.ts`:
       - TimerState interface: stepIndex (number|null), duration (number), startTimestamp (number|null), pausedRemaining (number|null), isRunning (boolean)
       - Hook returns: { timer, displaySeconds, start, pause, resume, reset }
       - `start(stepIndex, durationSeconds)`: sets new timer state, calls `ensureNotificationPermission()` then `scheduleTimerNotification(durationSeconds)`, stores notification ID in ref
       - On start, if previous notification exists, cancel it first (one timer at a time)
       - Tick interval: 200ms setInterval when isRunning, calculates remaining = duration - (Date.now() - startTimestamp) / 1000
       - AppState listener: on "active" event, timer auto-recalculates (no explicit action needed since displaySeconds derives from Date.now())
       - When remaining <= 0: clearInterval, set isRunning=false, call onComplete callback (passed to hook or exposed)
       - `pause()`: clear interval, store displaySeconds as pausedRemaining, cancel scheduled notification
       - `resume()`: restart from pausedRemaining as new duration, new startTimestamp = Date.now(), reschedule notification
       - `reset()`: clear all state, cancel notification
       - Cleanup on unmount: clear interval, cancel any scheduled notification
