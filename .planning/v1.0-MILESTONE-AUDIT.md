## Requirement Gaps

### DISC-03 — Advanced Filter Panel (Partial)

**Requirement:** User can filter recipes by craving/mood (cuisine type, cooking time, dish type — breakfast / soup / main / etc.)

**What works:** Category/dish-type chip filtering is wired. Tapping a category chip calls `filterRecipesByCategory()` and updates the displayed recipes correctly.

**What is broken:** The advanced filter panel (expanded via the ▼ arrow) exposes three additional filter dimensions:
- Cook time bucket (< 15 min, 15–30 min, > 30 min)
- Skill level filter
- Cuisine type

User interactions update `filter` state in React (`setFilter`) but `loadRecipes()` in `app/(tabs)/index.tsx` lines 113–140 only reads `filter.category` / `selectedCategory`. It never reads `filter.cookTimeBucket`, `filter.skillLevel`, or `filter.cuisine`, and never calls `queryRecipesByFilter`. The function `queryRecipesByFilter` is defined in `src/db/recipes.ts` lines 318–362 and exposed in `useRecipesDb()` at line 495, but is called from zero screens.

**Effect:** Selecting a cook time or cuisine filter appears to work (UI updates) but produces no change in displayed recipes.

**Fix:** In `loadRecipes` (or a separate `useEffect` on `filter`), detect non-default advanced filter values and call `queryRecipesByFilter(filter, profile.allergens)` instead of `getAllRecipesForFeed`.

---

### DISC-05 — Allergen Exclusion on Bookmarks (Partial)

**Requirement:** Allergen-incompatible recipes are automatically filtered out from all discovery surfaces.

**What works:** `ALLERGEN_EXCLUSION` SQL predicate applied in `getAllRecipesForFeed` and `getAllRecipesForSearch`. Feed and Search surfaces correctly hide allergen-incompatible recipes.

**What is broken:** `app/(tabs)/my-kitchen.tsx` executes a raw inline `db.getAllAsync()` query (lines 81–96) to fetch bookmarked recipes. This query does not include allergen exclusion predicates. `getAllRecipesForFeed` is imported at line 57 but never called.

**Effect:** A user who bookmarked a gluten-containing recipe before declaring a gluten allergen will continue to see that recipe in My Kitchen, even though they have the allergen set.

**Severity:** Low — this is an edge case (bookmarked before allergen was set) and does not affect new recipe discovery. The Fix is isolated to `my-kitchen.tsx`.

**Fix:** Either (a) add `ALLERGEN_EXCLUSION` predicate to the inline My Kitchen query, or (b) replace the inline query with a call to `getAllRecipesForFeed` and filter results to bookmarked IDs.
