## Phase Details

### Phase 13: Image Pipeline Foundation
**Goal**: Recipe images are optimized, bundled, and resolvable in production builds via a static registry
**Depends on**: Nothing (first phase of v1.1)
**Requirements**: IMG-04
**Success Criteria** (what must be TRUE):
  1. Running the build pipeline produces WebP images under 100KB each in the output directory
  2. A static TypeScript image registry maps recipe IDs to require() calls and is auto-generated by the build script
  3. YAML recipe files reference their cover image paths and the build script validates they exist
  4. A production release build (not just dev) correctly resolves bundled recipe images without errors
**Plans:** 2/2 plans complete

Plans:
- [ ] 13-01-PLAN.md — Build image pipeline script, registry generation, and menemen cover image
- [ ] 13-02-PLAN.md — Gitignore configuration and production export verification

### Phase 14: Color Token Sweep
**Goal**: Every color in the app flows through the theme system so dark mode contrast changes propagate everywhere
**Depends on**: Phase 13
**Requirements**: UX-01
**Success Criteria** (what must be TRUE):
  1. Zero hardcoded hex color values remain in component files outside theme.ts and intentional gradient palettes
  2. Recipe cards are clearly distinguishable from the background in dark mode (visible card boundary)
  3. Recipe cards are clearly distinguishable from the background in light mode
**Plans:** 3/3 plans complete

Plans:
- [ ] 14-01-PLAN.md — Expand theme tokens, create palette.ts, create test infrastructure
- [ ] 14-02-PLAN.md — Sweep recipe cards (UX-01 core) and core screens
- [ ] 14-03-PLAN.md — Sweep cooking flow, search, UI primitives, onboarding, and auth

### Phase 15: Card Image Rendering
**Goal**: Users see real food photography on recipe cards, recipe detail, and during cooking — the app looks like a finished product
**Depends on**: Phase 13 (image registry must exist), Phase 14 (theme tokens for gradient scrim colors)
**Requirements**: IMG-01, IMG-02, IMG-03
**Success Criteria** (what must be TRUE):
  1. Every recipe card in feed, search results, and cookbook shows a cover image (or graceful gradient fallback if no image exists)
  2. Each cooking mode step displays its step-specific image (or gradient fallback) above the step text
  3. A blurhash placeholder appears briefly while images load, instead of a blank space or layout jump
  4. Recipe detail screen shows the cover image as a hero at the top of the page
**Plans:** 3/3 plans complete

Plans:
- [ ] 15-01-PLAN.md — Blurhash pipeline: install blurhash, update build-images.ts, extend registry schema, add expo-image mock
- [ ] 15-02-PLAN.md — Card and detail images: wire expo-image into RecipeCardGrid, RecipeCardRow, and recipe detail hero
- [ ] 15-03-PLAN.md — Cooking step images: update StepContent to use registry-based expo-image with recipeId prop

### Phase 16: Feed "See All" + Discovery Polish
**Goal**: Users can browse full recipe lists per feed section and understand that horizontal sections are scrollable
**Depends on**: Phase 13
**Requirements**: DISC-06, DISC-07, DISC-08, DISC-09
**Success Criteria** (what must be TRUE):
  1. Tapping "See All" on any feed section navigates to a vertical scrollable list showing all recipes in that section
  2. A partial third card is visible at the right edge of each feed section, hinting that more content exists
  3. Feed sections show a subtle animation on first appearance that suggests horizontal scrolling
  4. Each feed section heading displays the recipe count (e.g., "Sana Ozel (8)") with a visual separator between sections
**Plans**: TBD

### Phase 17: Cookbook Saved/Cooked Tabs
**Goal**: Users can revisit their cooking history with ratings and see their saved recipes in a clean tabbed layout
**Depends on**: Phase 15 (images on cards)
**Requirements**: BOOK-01, BOOK-02, BOOK-03, BOOK-04, UX-08
**Success Criteria** (what must be TRUE):
  1. Cookbook screen has two visible tabs — Saved (bookmarked recipes) and Cooked (cooking history) — and switching between them is instant
  2. Each recipe in the Cooked tab shows its star rating from when the user completed it
  3. User can tap a rating in the Cooked tab to change it and the new rating persists across app restarts
  4. Cooked recipes display a cook count (e.g., "3 kez pisirdin") when cooked more than once
  5. Cookbook uses a single-recipe-per-row layout with thumbnail, title, and metadata on each row
**Plans**: TBD

### Phase 18: UI Polish
**Goal**: Every interaction in the app feels responsive and polished through haptics, animations, and visual feedback
**Depends on**: Phase 15, Phase 16, Phase 17 (polish applies to completed features)
**Requirements**: UX-02, UX-03, UX-04, UX-05, UX-06, UX-07
**Success Criteria** (what must be TRUE):
  1. Bookmarking a recipe plays a heart animation and the user feels a haptic tap
  2. Tapping any recipe card shows a subtle press-down scale effect before navigation
  3. Bottom sheets (Malzemeler, Sefim) fade their backdrop in and out smoothly instead of appearing/disappearing instantly
  4. Search filter chips display a small category icon next to their label
  5. The cooking mode progress bar animates smoothly when moving between steps instead of jumping
  6. Buttons across the app provide visual feedback on tap (scale or opacity change)
**Plans**: TBD
